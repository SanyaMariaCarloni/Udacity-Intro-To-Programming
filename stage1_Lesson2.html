Lesson 1: Basics of the Web & HTML

What is The WWW?

The WWW, or more formally World Wide Web, is comprised of various documents. These documents range from videos, photos, audio files and text documents. The web is essentially considered cloudbased because it gets to us through the airwaves but it is not invisible, really. The web is really a mass of computer servers that communicate with one another. In order for it to communicate properly, it needs the help of Hypertext Transfer Protocol, HTTP, and browsers (or clients) to fully function. Again, it is through the Hypertext Transfer Protocol, that facilitates communication between the client and servers.

Are Computers Really Stupid?

Computers lack the ability, unless programmed, to make decisions on their own. Computers rely on us to tell them what to do. They are a tool for our use to produce and distribute digital information. So, a tiny mistake in code by a programmer can cause a program to not function properly.

HTML

Hypertext Markup Language is a very broad markup which we use to tell computers to do things, specifically with webpages. We want the computer to know how we want to add context and structure so with HTML you can create this! It is made up of:
  • text content - what you see
  • markup - what it looks like
  • references - to other documents such as images and video
  • links - to other pages

All of the above are created by tags with emphasis on certain elements and attributes.

Tag names are always the same except the closing tag has a slash.
For example: <name>text/content</name>

Aditionaly, we have:

  • <span> -- inline
  • <div> -- block

Elements add structure to the HTML document where as Attributes provide additional information to the browser.

Whitespaces

Whitespaces render multiple lines; line break.
We have inline and block. The difference between the two are self-explanatory, to many. Inline keeps text/content on the same line and appears, well, in line; whereas block makes text/content appear within an invisible box.
Some examples of block element tags are: <p>, <div>,<h1>. Here is a quick video recapping whitespaces in HTML.

Developer Tools within a Browser (specifically Google Chrome)

HTML elements can be manipulated within the devloper tools of a browser. With Google Chrome, you can adjust certain elements to see their differneced but can not save to the actual .html file. Whatever adjustments are made within developer tools, has to be adjusted within the actual HTML document.

Tree-like Structure in HTML

A tree-like structure is a way of sectioning data in a heirarchy. There are the tree elements, that resemble the base of the tree, called nodes; connecting elements which act as the branches and finally end-nodes, which have no data associated to them.
The back-end shows a wave of symbols and letters. Whereas, in the front-end, the information/text/content/data displayed appears as content within invisible (or visible with CSS affixed) boxes. These boxes are essentially caccooned within other boxes... Kind of reminds me of the Fibonacci Sequence.

HTML -- CSS -- DOM

HTML defines the content and structure of a page.

CSS defines the style. It allows the programmer to use specific syntax and rules to change how elements look on a page; the size, font, color, background and border.

DOM is the Document Object Model and tree-like structure.
